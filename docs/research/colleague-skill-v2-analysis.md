## 一、核心定位

### 1.1 论文解决的核心问题

> **人的专业知识、判断力和交互风格散布在异构痕迹中（代码注释、聊天记录、会议纪要、邮件等），现有系统无法将这些痕迹转化为可检查、可纠正、可移植的 AI 技能包。**

### 1.2 核心洞察

**不是"数字分身"，而是"可检查的技能制品"：**

```
传统做法：人 → 隐藏记忆/不透明prompt → 模仿行为
COLLEAGUE.SKILL：人 → 异构痕迹 → 双轨蒸馏 → 版本化技能包
                                              ├── 可检查（文件级可见）
                                              ├── 可纠正（自然语言反馈）
                                              ├── 可回滚（版本历史）
                                              ├── 可移植（跨Agent宿主安装）
                                              └── 可治理（来源边界、权限控制）
```

### 1.3 五大产品特性

| 特性 | 含义 | 对Linkedevery的意义 |
|------|------|-------------------|
| **Portable** | 技能包可被任何Skills兼容的Agent加载 | 导师OPC可在不同平台运行 |
| **Inspectable** | 用户可读取提取的规则、示例、限制 | 企业可审查导师能力声明 |
| **Composable** | 全量/仅能力/仅风格 可分别调用 | 企业可选择"只用能力"或"能力+风格" |
| **Correctable** | 新证据或反馈可更新技能包 | 导师可纠正蒸馏结果，持续优化 |
| **Governable** | 元数据、来源边界、免责声明支持治理 | 数据资产确权、权限控制 |

## 二、技术架构

### 2.1 系统总览

```
COLLEAGUE.SKILL 蒸馏Pipeline

1. 痕迹采集        2. 预设路由        3. 双轨蒸馏        4. 制品写入        5. 产品化
docs/email/截图    colleague核心      能力轨道            SKILL.md           Agent宿主
chats/评审/公开    relationship扩展    人格轨道            manifest.json      可选Gallery
                   celebrity扩展       work.md+persona.md  meta.json          安装/调用/分享

治理轨道：
本地优先存储 → 来源+证据 → 纠正日志 → 版本/回滚 → 可选Gallery
```

### 2.2 三大应用预设（Presets）

| 预设 | 证据来源 | 治理要求 | 运行时 | 调用别名 |
|------|---------|---------|--------|---------|
| **colleague** | 企业/本地工作痕迹：文档、评审、决策、事件 | 组织访问权限、工作交接 | 实用导向，工作专用入口 | `/create-colleague` |
| **celebrity** | 公开来源：第一人称作品、访谈、演讲 | 来源边界、引用纪律 | 研究导向，可见证据限制 | `/create-icon` |
| **relationship** | 用户提供的私人交互痕迹 | 同意、本地控制、删除、非公开默认 | 本地可编辑状态 | `/create-ex` |

**关键设计：** 三个预设共享同一蒸馏流水线，只是证据范围、治理要求、调用语义不同。新增预设（如 self、author、team）只需配置和prompt调整。

### 2.3 双轨表示（Dual Representation）

```
技能包 = 能力轨道（work.md）+ 行为轨道（persona.md）

能力轨道（Part A）：
├── 职责、工作流、技术标准
├── 评审标准、决策启发式
├── 过往经验教训
└── 任务模式

行为轨道（Part B）：
├── 表达偏好、交互姿态
├── 行为边界、纠正历史
└── 交互规则

运行时规则：
Agent选择相关行为约束 → 应用能力/心智模型轨道 → 产出在制品边界内的响应
```

**为什么要分轨？** 因为人格系统的常见失败来自混淆三件事：事实知识、程序判断、表面风格。COLLEAGUE.SKILL 让这三者可检查、可分别调用。

### 2.4 制品契约（Artifact Contract）

| 制品 | 主要消费者 | 内容 |
|------|-----------|------|
| **SKILL.md** | Agent运行时、用户 | 组合可调用技能（frontmatter + 能力轨道 + 行为轨道 + 操作规则） |
| **work.md** | 用户、更新者 | 可编辑能力文档：流程、标准、启发式、任务模式 |
| **persona.md** | 用户、更新者 | 可编辑行为文档：风格、交互姿态、边界、纠正日志 |
| **work_skill.md** | Agent运行时 | 从work.md生成的仅能力入口 |
| **persona_skill.md** | Agent运行时 | 从persona.md生成的仅行为入口 |
| **manifest.json** | 安装器、Gallery | 入口、制品列表、兼容运行时、斜杠命令、工具链元数据 |
| **meta.json** | 生命周期工具 | Schema、来源、生命周期版本、纠正计数、兼容性字段 |

## 三、蒸馏与进化工作流

### 3.1 创建工作流

```
用户提供：别名 + 可选画像字段 + 源材料
    ↓
采集器/解析器：飞书/钉钉/Slack/微信SQLite/邮件/PDF/截图/Markdown
    ↓
归一化为本地知识目录
    ↓
分析器：提取持久能力、心智模型、有界交互风格
    ↓
构建器：渲染结构化Markdown
    ↓
写入器：打包为版本化技能包（Schema v3）
```

### 3.2 纠正与更新工作流

```
用户反馈："他不会这么说" / "她会在这里反驳"
    ↓
纠正处理器识别自然语言反馈
    ↓
├── 能力相关 → Markdown补丁（按二级标题匹配替换，未匹配追加）
└── 行为相关 → 标准化纠正记录 {scene, wrong, correct}
    ↓
写入器：归档当前版本 → 应用补丁/纠正 → 递增版本号 → 重新生成所有派生制品
    ↓
版本管理器：列出归档版本 / 备份当前 / 回滚到旧版本 / 清理旧归档
```

### 3.3 版本化技能生命周期

```
创建 → 检查 → 躺平 → 纠正 → 归档 → 安装/分享
v1包    来源+制品  全量/工作/人格  补丁或行为记录  v2+先前状态  宿主或Gallery
                                    ↓
                              纠正记录：scene/wrong/correct
                              补丁+重新生成：section merge + 派生文件
                              归档元数据：version, provenance, count
                              回滚/删除：恢复或移除技能包
                              用户拥有的状态：可审计历史
```

## 四、名人研究扩展（Celebrity Preset）

### 4.1 六维研究流程

名人预设强调**第一人称作品、长访谈、有记录的决策、明确标记的推断**。工具链包括：
- 字幕下载、音频转录、字幕清理
- 研究笔记合并、质量检查

### 4.2 质量检查器扫描项

- 心智模型覆盖率
- 限制/边界
- 表达模式
- 内部张力
- 来源URL
- 版权安全信号

**关键：** 不认证事实真相，而是记录证据限制，在证据薄弱时降级置信度。

## 五、关系扩展（Relationship Preset）

### 5.1 敏感私人领域

- 价值不是"Agent替代人"，而是"个人交互痕迹可表示为本地、可编辑、可删除的状态"
- 相比colleague和celebrity，需要更强的同意、保留、访问控制假设

### 5.2 风险识别

- 情感过度依附
- 未经同意的模拟
- 私人聊天滥用

### 5.3 部署优先级

- 本地所有权、删除、明确免责声明、opt-in分享

### 7.1 直接技术复用

| COLLEAGUE.SKILL能力 | Linkedevery应用 |
|-------------------|----------------|
| 双轨蒸馏（能力+行为） | 导师OPC的L2/L3蒸馏直接采用 |

| 多宿主安装 | 导师OPC可在不同企业平台运行 |

### 7.2 蒸馏Pipeline映射

```
COLLEAGUE.SKILL                    Linkedevery
─────────────────                  ─────────────────
痕迹采集                           数据采集（A1爬虫 + 导师贡献）
  ↓                                  ↓
预设路由                           按行业/学科路由到不同蒸馏prompt
  ↓                                  ↓
双轨蒸馏                           L1标签画像 + L2能力提取 + L3风格建模
  ↓                                  ↓
制品写入                           SKILL.md生成（导师OPC能力声明）
  ↓                                  ↓

  ↓                                  ↓
```

### 7.3 三大预设在Linkedevery中的对应

| COLLEAGUE.SKILL预设 | Linkedevery场景 | 证据来源 |
|-------------------|----------------|---------|
| **colleague** | 导师→企业技术对接 | 论文、项目、案例、评审记录 |
| **celebrity** | 行业专家公开知识 | 公开演讲、访谈、著作 |
| **relationship** | 长期导师-企业陪跑 | 交互历史、反馈记录 |

### 7.4 关键差异化机会

**COLLEAGUE.SKILL 做了蒸馏，但没做交易。Linkedevery 的独特价值：**

```
COLLEAGUE.SKILL：痕迹 → 技能包 → 安装/分享（免费）
Linkedevery：    痕迹 → 技能包 → 能力资产化 → TOKEN定价 → 交易（商业）
                                                      ↑
                                              区块链确权 + 智能合约
```

**核心壁垒：** 不是"又一个蒸馏工具"，而是"蒸馏+确权+交易"的完整闭环。

### 7.5 Phase 1 技术方案调整建议

| 原方案 | 调整建议 | 理由 |
|--------|---------|------|
| L2蒸馏自研 | 直接复用COLLEAGUE.SKILL Pipeline | 开源、成熟、社区活跃 |
| 技能包格式自定义 | 采用Agent Skills标准（SKILL.md） | 兼容Claude/OpenClaw/Hermes |
| 版本管理自研 | 复用COLLEAGUE.SKILL版本管理 | 纠正/回滚/归档机制完善 |
| 蒸馏质量验证 | 引入COLLEAGUE.SKILL质量检查器 | 心智模型覆盖率、证据边界 |

### 7.6 蒸馏定位再确认

上次共识：**"智能助理"而非"数字分身"**

COLLEAGUE.SKILL V2 的表述更精确：

> **"Bounded person-grounded artifact"** — 有界的人格锚定制品  
> 不是身份替代，不是同意代理，而是可检查、可纠正、可治理的技术制品

建议Linkedevery采用这个更精确的定位话术。

## 八、论文的局限性与负责任部署

### 8.1 论文承认的局限

- 不声称生成的技能忠实地复现了一个人
- 来源匹配、任务性能、情感安全、用户信任校准仍是开放问题
- 纠正可以改善制品，但也可能编码编辑偏差
- 真实部署依赖于来源质量、提取质量、模型行为、人工审核

### 8.2 负责任部署要求

- 明确参与、有界来源收集、访问控制、保留限制、非强制使用
- 本地优先、可检查、版本化设计提供治理便利
- Gallery发布应为opt-in，提交者证明、审核、下架、来源边界标签、可见免责声明

## 九、学术定位：与现有工作的区别

| 相关工作 | COLLEAGUE.SKILL的区别 |
|---------|---------------------|
| Voyager（技能库积累） | 从人类痕迹蒸馏，不是从执行轨迹 |
| SkillX/SkillGen（轨迹蒸馏） | 分离能力和有界行为，暴露纠正/回滚状态 |
| AutoSkill（交互抽象） | 面向人格锚定，不是通用技能 |
| Character-LLM/RoleLLM（角色扮演） | 更窄：构建显式可审查制品，非开放模拟 |
| RAG/个性化（记忆系统） | 不是隐藏记忆，是可检查文件 |

## 十、总结

### COLLEAGUE.SKILL V2 的核心贡献

1. **问题定义：** 将"人格锚定技能生成"定义为制品问题（可移植、可检查、可组合、可纠正、可治理）
2. **系统实现：** 双轨蒸馏Pipeline + 三大预设 + 版本化生命周期
3. **社区验证：** 18.5k stars, 215 skills, 165 contributors
4. **开放问题：** 行为保真度边界、任务性能评估、情感安全

### 对Linkedevery的一句话总结

> **COLLEAGUE.SKILL 解决了"怎么蒸馏"，Linkedevery 要解决"蒸馏后怎么交易"。直接复用COLLEAGUE.SKILL的蒸馏Pipeline，叠加区块链确权+TOKEN经济+智能合约，就是Linkedevery的核心技术路线。**
