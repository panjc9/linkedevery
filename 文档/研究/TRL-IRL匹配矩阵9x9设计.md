### 1.2 匹配规则说明

| 符号 | 含义 | TRL-IRL差距 | 说明 |
|------|------|-------------|------|
| ✅ | 最佳匹配 | TRL - IRL = 1~2 | 导师能带企业升级 |
| ⚠️ | 可用匹配 | TRL - IRL = 0 或 3~4 | 同级互助或差距大 |
| ❌ | 不匹配 | TRL - IRL < 0 或 > 4 | 导师太落后或太超前 |

## 二、四个等级定义

### 2.1 四个等级概览

| 等级 | 英文 | 核心需求 | 企业特征 | 匹配重点 |
|------|------|----------|----------|----------|
| **服务型** | Service | 快速解决问题 | 有明确痛点，需要即时帮助 | 响应速度、问题解决能力 |
| **工程型** | Engineering | 项目交付 | 有具体项目，需要完整交付 | 项目管理、交付能力 |
| **技术型** | Technology | 技术突破 | 有技术难题，需要创新能力 | 技术深度、创新能力 |
| **研究型** | Research | 知识探索 | 有研究需求，需要前沿探索 | 学术能力、研究深度 |

### 2.2 服务型（L1）

**定义**：企业有明确痛点，需要快速解决问题

**企业特征**：
- 问题明确，解决方案相对标准化
- 需要快速响应，时间紧迫
- 预算有限，追求性价比
- 技术基础一般，需要简单易用的方案

**匹配重点**：
- **响应速度**：导师能快速响应企业需求
- **问题解决能力**：导师有类似问题的解决经验
- **沟通能力**：导师能清晰解释解决方案
- **成本控制**：导师能在预算内提供服务

**TRL/IRL匹配范围**：
- 导师TRL：T4-T6（实验室验证到小批量）
- 企业IRL：IRL1-IRL3（识别到验证）
- 最佳差距：TRL - IRL = 1~2

**示例场景**：
- 企业需要修复一个生产故障
- 企业需要优化一个现有流程
- 企业需要培训员工使用新工具

---

### 2.3 工程型（L2）

**定义**：企业有具体项目，需要完整交付

**企业特征**：
- 有明确的项目目标和交付物
- 有一定技术基础，需要专业指导
- 预算充足，追求质量和效果
- 需要项目管理和风险控制

**匹配重点**：
- **项目管理能力**：导师有项目管理经验
- **交付能力**：导师有成功交付案例
- **技术能力**：导师有相关技术栈经验
- **风险控制**：导师能识别和控制项目风险

**TRL/IRL匹配范围**：
- 导师TRL：T5-T7（中试验证到批量）
- 企业IRL：IRL2-IRL4（定义到描述）
- 最佳差距：TRL - IRL = 1~3

**示例场景**：
- 企业要开发一个新系统
- 企业要实施一个数字化转型项目
- 企业要建设一个数据平台

---

### 2.4 技术型（L3）

**定义**：企业有技术难题，需要创新能力

**企业特征**：
- 遇到技术瓶颈，需要突破
- 有一定技术团队，需要高端指导
- 预算充足，追求技术领先
- 需要创新思维和前沿技术

**匹配重点**：
- **技术深度**：导师在相关领域有深厚积累
- **创新能力**：导师有创新思维和方法
- **研发能力**：导师有研发经验和成果
- **技术视野**：导师了解技术发展趋势

**TRL/IRL匹配范围**：
- 导师TRL：T6-T8（小批量到规模化）
- 企业IRL：IRL3-IRL6（验证到真实环境）
- 最佳差距：TRL - IRL = 2~4

**示例场景**：
- 企业要攻克一个技术难题
- 企业要开发一个创新产品
- 企业要建立技术壁垒

---

### 2.5 研究型（L4）

**定义**：企业有研究需求，需要前沿探索

**企业特征**：
- 有研究部门，需要学术指导
- 追求技术领先，需要前沿探索
- 预算充足，追求长期价值
- 需要学术资源和研究网络

**匹配重点**：
- **学术能力**：导师有学术背景和成果
- **研究能力**：导师有研究经验和方法
- **前沿视野**：导师了解技术前沿趋势
- **学术网络**：导师有学术资源和人脉

**TRL/IRL匹配范围**：
- 导师TRL：T7-T9（批量到产业化）
- 企业IRL：IRL5-IRL9（相关环境到可持续）
- 最佳差距：TRL - IRL = 3~5

**示例场景**：
- 企业要建立研究院
- 企业要开展前沿技术研究
- 企业要发表学术论文

## 四、匹配算法实现

### 4.1 需求等级识别

```python
def identify_requirement_level(requirement):
    """
    识别企业需求等级
    返回：L1/L2/L3/L4
    """
    # 1. 分析问题类型
    if requirement.problem_type == "快速解决":
        return "L1"  # 服务型
    elif requirement.problem_type == "项目交付":
        return "L2"  # 工程型
    elif requirement.problem_type == "技术突破":
        return "L3"  # 技术型
    elif requirement.problem_type == "知识探索":
        return "L4"  # 研究型
    
    # 2. 分析企业特征
    if requirement.urgency == "HIGH" and requirement.budget_level == "LOW":
        return "L1"  # 服务型
    elif requirement.project_scope == "明确" and requirement.team_size > 10:
        return "L2"  # 工程型
    elif requirement.tech_difficulty == "HIGH" and requirement.innovation_need == "HIGH":
        return "L3"  # 技术型
    elif requirement.research_orientation == "HIGH" and requirement.academic_collaboration == "YES":
        return "L4"  # 研究型
    
    # 默认返回服务型
    return "L1"
```

### 4.2 根据需求等级筛选导师

```python
def filter_mentors_by_level(mentors, requirement_level):
    """
    根据需求等级筛选导师
    """
    if requirement_level == "L1":  # 服务型
        # 导师TRL范围：T4-T6
        return [m for m in mentors if 4 <= m.trl.level <= 6]
    
    elif requirement_level == "L2":  # 工程型
        # 导师TRL范围：T5-T7
        return [m for m in mentors if 5 <= m.trl.level <= 7]
    
    elif requirement_level == "L3":  # 技术型
        # 导师TRL范围：T6-T8
        return [m for m in mentors if 6 <= m.trl.level <= 8]
    
    elif requirement_level == "L4":  # 研究型
        # 导师TRL范围：T7-T9
        return [m for m in mentors if 7 <= m.trl.level <= 9]
    
    return mentors
```

### 4.3 计算匹配得分

```python
def calculate_match_score(mentor, enterprise, requirement_level):
    """
    计算匹配得分
    """
    mentor_trl = mentor.trl.level
    enterprise_irl = enterprise.irl.level
    gap = mentor_trl - enterprise_irl
    
    # 根据需求等级确定最佳距离范围
    if requirement_level == "L1":  # 服务型
        ideal_min, ideal_max = 1, 2
        tolerance_min, tolerance_max = 0, 3
    elif requirement_level == "L2":  # 工程型
        ideal_min, ideal_max = 1, 3
        tolerance_min, tolerance_max = 0, 4
    elif requirement_level == "L3":  # 技术型
        ideal_min, ideal_max = 2, 4
        tolerance_min, tolerance_max = 1, 5
    elif requirement_level == "L4":  # 研究型
        ideal_min, ideal_max = 3, 5
        tolerance_min, tolerance_max = 2, 6
    
    # 计算距离得分
    if ideal_min <= gap <= ideal_max:
        # 最佳区间
        distance_score = 1.0
    elif tolerance_min <= gap <= tolerance_max:
        # 可接受范围
        distance_score = 0.6 + 0.4 * (1 - abs(gap - (ideal_min + ideal_max) / 2) / (tolerance_max - tolerance_min))
    else:
        # 不匹配
        distance_score = 0.1
    
    # 置信度惩罚
    trl_confidence = mentor.trl.confidence
    irl_confidence = enterprise.irl.confidence
    confidence_penalty = min(trl_confidence, irl_confidence)
    
    # 最终得分
    score = distance_score * confidence_penalty
    
    return score
```

## 六、总结

### 6.1 核心设计原则

1. **9×9匹配矩阵**：TRL 9级 × IRL 9级，覆盖所有匹配场景
2. **四个等级**：服务型、工程型、技术型、研究型，满足不同企业需求
3. **动态匹配范围**：根据需求等级动态调整TRL/IRL匹配范围
4. **最佳带教距离**：导师TRL领先企业IRL 1-2级为最佳

### 6.2 匹配规则总结

| 等级 | 导师TRL | 企业IRL | 最佳差距 | 可接受差距 |
|------|---------|---------|----------|------------|
| 服务型 | T4-T6 | IRL1-IRL3 | 1-2 | 0-3 |
| 工程型 | T5-T7 | IRL2-IRL4 | 1-3 | 0-4 |
| 技术型 | T6-T8 | IRL3-IRL6 | 2-4 | 1-5 |
| 研究型 | T7-T9 | IRL5-IRL9 | 3-5 | 2-6 |

### 6.3 设计亮点

1. **完整性**：9×9矩阵覆盖所有TRL/IRL组合
2. **灵活性**：四个等级适应不同企业需求
3. **科学性**：动态匹配范围基于最佳带教距离理论
4. **可操作性**：清晰的匹配规则和算法实现
