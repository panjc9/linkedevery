# TRL/IRL 匹配算法权重调整说明

**调整日期**：2026-06-16  
**调整人**：海猪（产品负责人）  
**调整原因**：行业匹配度权重应为最大

---

## 一、调整背景

在评审TRL/IRL匹配算法技术架构时，海猪指出：**行业匹配度权重应该最大**。

**理由**：
1. 行业经验决定匹配质量 - 导师的行业经验直接影响指导效果
2. 行业痛点理解 - 只有同行业导师才能真正理解企业痛点
3. 案例可复用 - 同行业导师的成功案例更有参考价值
4. 客户信任度 - 企业更愿意选择有同行业经验的导师

---

## 二、权重调整对比

### 调整前（原设计）

| 维度 | 权重 | 说明 |
|------|------|------|
| TRL/IRL 适配度 | 35% | 与需求等级、最佳带教距离直接相关 |
| 技能对齐度 | 25% | 导师技能与企业需求主题匹配 |
| **行业匹配度** | **15%** | 行业标签、子行业、产业链位置匹配 |
| 规模适配度 | 10% | 企业规模与导师服务方式匹配 |
| 服务形式 | 10% | 陪跑/咨询/培训等服务形式匹配 |
| 地域时间 | 5% | 城市、通勤、响应周期匹配 |

### 调整后（新设计）

| 维度 | 权重 | 调整幅度 | 说明 |
|------|------|----------|------|
| **行业匹配度** | **35%** | +20% | 行业经验决定匹配质量 |
| TRL/IRL 适配度 | 25% | -10% | 技术能力匹配 |
| 技能对齐度 | 20% | -5% | 技能需求匹配 |
| 规模适配度 | 10% | 0% | 保持不变 |
| 服务形式 | 5% | -5% | 可协商调整 |
| 地域时间 | 5% | 0% | 保持不变 |

## 三、调整后的匹配算法

### 3.1 总公式

```
final_score(M, E, R) =
    w1 * s_industry(M, E)           // 行业匹配度 35%
  + w2 * s_trl_irl(M, E, R)         // TRL/IRL适配度 25%
  + w3 * s_skill(M, E, R)           // 技能对齐度 20%
  + w4 * s_scale(M, E)              // 规模适配度 10%
  + w5 * s_service(M, E, R)         // 服务形式 5%
  + w6 * s_geo_time(M, E)           // 地域时间 5%

其中：
w1=0.35, w2=0.25, w3=0.20, w4=0.10, w5=0.05, w6=0.05
```

### 3.2 行业匹配度评分算法

```python
def score_industry(mentor, enterprise):
    """
    计算行业匹配度（权重35%）
    基于导师行业经验与企业所属行业的匹配程度
    """
    # 1. 精确匹配（同行业）
    exact_match = exact_tag_overlap(
        mentor.good_at_industries, 
        enterprise.industry_tags
    )
    
    # 2. 相关行业匹配（产业链上下游）
    related_match = related_industry_graph_score(
        mentor.good_at_industries, 
        enterprise.industry_tags
    )
    
    # 3. 行业经验深度
    industry_depth = mentor.industry_experience_years / 10  # 10年为满分
    
    # 4. 行业案例数量
    case_count_score = min(mentor.industry_cases_count / 10, 1.0)  # 10个案例为满分
    
    # 综合评分
    score = (
        0.4 * exact_match +           # 精确匹配 40%
        0.3 * related_match +         # 相关行业 30%
        0.2 * industry_depth +        # 行业深度 20%
        0.1 * case_count_score        # 案例数量 10%
    )
    
    return score
```

## 四、IRL 九级体系说明

### 4.1 原始IRL五级体系

| 等级 | 定义 |
|------|------|
| L1 | 概念级 - 识别集成需求 |
| L2 | 原型级 - 定义集成方案 |
| L3 | 试点级 - 验证可行性 |
| L4 | 推广级 - 规模化集成 |
| L5 | 成熟级 - 持续优化 |

### 4.2 创新的IRL九级体系

| 等级 | 定义 | 示例 |
|------|------|------|
| IRL 1 | 集成方案被识别 | 企业意识到需要引入AI质检，开始调研 |
| IRL 2 | 集成方案被定义 | 确定AI质检系统与现有MES/ERP的接口方式 |
| IRL 3 | 集成方案可行性被验证 | AI质检与MES系统在测试环境中成功对接 |
| IRL 4 | 集成技术被充分描述 | 完成接口文档、数据字典、异常处理规范 |
| IRL 5 | 集成方案在相关环境中被验证 | AI质检在模拟产线上运行，与MES实时同步 |
| IRL 6 | 集成方案在真实环境中被验证 | AI质检在客户工厂实际产线上运行1个月 |
| IRL 7 | 集成方案被完整集成 | AI质检成为产线的标准环节，自动触发/报警 |
| IRL 8 | 集成方案被验证为可靠 | AI质检系统运行6个月，准确率>99% |
| IRL 9 | 集成方案被证明为可持续 | AI质检系统已推广到所有产线，纳入质量管理体系 |

### 4.3 五级→九级映射关系

| 原始五级 | 对应九级 | 升级说明 |
|----------|----------|----------|
| L1 概念级 | IRL 1-2 | 概念识别 + 方案定义 |
| L2 原型级 | IRL 3-4 | 可行性验证 + 技术描述 |
| L3 试点级 | IRL 5-6 | 相关环境验证 + 真实环境验证 |
| L4 推广级 | IRL 7-8 | 完整集成 + 可靠性验证 |
| L5 成熟级 | IRL 9 | 可持续运营 |

## 六、总结

**权重调整核心变化**：
- 行业匹配度：15% → 35%（+20%）
- TRL/IRL适配度：35% → 25%（-10%）
- 技能对齐度：25% → 20%（-5%）
- 服务形式：10% → 5%（-5%）

**IRL九级体系价值**：
- 更精细的评估（5级→9级）
- 更清晰的升级路径
- 更准确的匹配结果
- 更好的进度跟踪
