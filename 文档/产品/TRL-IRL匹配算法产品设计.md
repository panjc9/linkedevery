## 一、产品概述

### 1.1 产品定位

LinkedEvery 匹配算法产品是智能体中台的核心模块，基于 **TRL（技术成熟度等级）× IRL（集成成熟度等级）** 双维度模型，实现导师与企业的精准匹配。产品核心价值：

- **不是"找相同"，而是"诊断→处方→配药"**
- **最佳带教距离：导师 TRL 领先企业 IRL 1-2 级**
- **服务价格 = 基础价格 × 需求等级系数 × TRL 稀缺性系数 × IRL 差距系数**

### 1.2 目标用户

| 用户角色 | 核心诉求 | 使用频率 |
|---------|---------|---------|
| **导师** | 获取精准客户、展示专业能力、提升服务效率 | 日常 |
| **企业** | 找到合适导师、降低试错成本、加速数字化转型 | 按需 |
| **运营人员** | 监控匹配效果、优化算法、管理用户 | 日常 |
| **客户成功经理** | 跟踪服务效果、提升客户满意度、推动续费 | 周/月 |

## 二、用户场景分析

### 2.1 导师使用场景

#### 场景一：导师入驻与信息录入

**用户故事：** 作为导师，我希望快速录入我的专业信息，让平台了解我的能力，以便匹配到合适的企业。

**关键流程：**

```
注册登录 → 基础信息填写 → TRL证据链上传 → 技能标签选择 → 平台审核 → 入驻成功
    │            │               │               │              │
    ▼            ▼               ▼               ▼              ▼
 手机/微信    姓名/单位/      论文/专利/       行业/技能/     1-3工作日
  快速注册    研究方向        项目案例/        服务类型        人工+AI
                            产业化经验                       自动审核
```

**TRL 四类证据链录入界面：**

| 证据类型 | 录入方式 | 验证机制 | 对应TRL区间 |
|---------|---------|---------|------------|
| **学术成果** | 自动抓取（知网/Google Scholar）+ 手动补充 | DOI/论文链接自动验证 | T1-T3 |
| **专利成果** | 专利号自动查询 + 手动补充 | 国家专利局API验证 | T3-T5 |
| **项目案例** | 结构化表单（项目名称/规模/成果/客户） | 客户确认/合同上传 | T5-T7 |
| **产业化经验** | 结构化表单（产品/营收/客户数/规模化程度） | 工商信息/财报佐证 | T7-T9 |

**界面设计要点：**
- **智能预填：** 输入姓名+单位后，自动抓取公开学术/专利信息，减少手动输入
- **分步引导：** 采用分步表单，每步3-5个字段，避免信息过载
- **实时预览：** 右侧实时显示"导师能力画像"，让导师看到自己被如何描述
- **进度提示：** 显示"信息完整度"百分比，激励导师补充更多资料

- **降低焦虑：** 用通俗语言解释专业术语，配以行业案例

#### 3.1.2 TRL 自动评估逻辑

| 证据类型 | 评估规则 | 权重 |
|---------|---------|------|
| **学术成果** | 论文数量×期刊等级 + 引用数 | 25%（T1-T3贡献） |
| **专利成果** | 发明专利数×转化率 + 实用新型数 | 25%（T3-T5贡献） |
| **项目案例** | 项目数×规模系数×客户评价 | 30%（T5-T7贡献） |
| **产业化经验** | 产品营收×客户数×规模化程度 | 20%（T7-T9贡献） |

**自动评估公式：**
```
导师TRL = weighted_max(
    学术贡献分 → T1-T3,
    专利贡献分 → T3-T5,
    案例贡献分 → T5-T7,
    产业化贡献分 → T7-T9
)
```

### 7.1 核心风险

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|---------|
| **数据冷启动** | 初期匹配质量差 | 高 | 种子用户邀请 + 人工匹配辅助 |

| **企业配合度低** | IRL数据不完整 | 中 | 简化问卷 + 激励机制 |
| **算法偏见** | 匹配不公平 | 中 | A/B测试 + 人工审核 |
| **竞品模仿** | 市场竞争 | 低 | 持续迭代 + 数据壁垒 |
