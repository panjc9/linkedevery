## 一、产品概述

### 1.1 产品定位

LinkedEvery 匹配算法产品是智能体中台的核心模块，基于 **TRL（技术成熟度等级）× IRL（集成成熟度等级）** 双维度模型，实现导师与企业的精准匹配。产品核心价值：

- **不是"找相同"，而是"诊断→处方→配药"**
- **最佳带教距离：导师 TRL 领先企业 IRL 1-2 级**
- **15个交叉点类型**：实现更精准的匹配和定价
- **六维度匹配模型**：行业30% + 技能20% + TRL/IRL20% + 规模10% + 服务10% + 地域10%

### 1.2 目标用户

| 用户角色 | 核心诉求 | 使用频率 |
|---------|---------|---------|
| **导师** | 获取精准客户、展示专业能力、提升服务效率 | 日常 |
| **企业** | 找到合适导师、降低试错成本、加速数字化转型 | 按需 |
| **运营人员** | 监控匹配效果、优化算法、管理用户 | 日常 |
| **客户成功经理** | 跟踪服务效果、提升客户满意度、推动续费 | 周/月 |

## 三、六维度匹配模型

### 3.1 权重设计

| 维度 | 权重 | 说明 |
|------|------|------|
| **行业匹配度** | **30%** | 行业标签、子行业、产业链位置匹配（第一位） |
| **技能对齐度** | **20%** | 导师技能与企业需求主题、CDIO能力匹配（第二位） |
| **TRL/IRL 适配度** | **20%** | 与需求等级、最佳带教距离直接相关（第三位） |
| 规模适配度 | 10% | 企业规模与导师服务方式、组织复杂度匹配 |
| 服务形式 | 10% | 陪跑/咨询/培训/方案评审等服务形式匹配 |
| 地域时间 | 10% | 城市、通勤、响应周期、项目时长匹配 |

**验证**：前三维度权重 = 30% + 20% + 20% = 70% > 66.67% ✅

### 3.2 匹配得分计算

```python
def calculate_match_score(mentor, enterprise, requirement):
    """
    计算匹配得分（六维度）
    """
    # 1. 识别匹配类型
    match_type = identify_match_type(mentor.trl.level, enterprise.irl.level)
    
    # 2. 六维度评分
    industry_score = score_industry(mentor, enterprise)  # 30%
    skill_score = score_skill(mentor, enterprise, requirement)  # 20%
    trl_irl_score = score_trl_irl(mentor, enterprise, requirement)  # 20%
    scale_score = score_scale(mentor, enterprise)  # 10%
    service_score = score_service(mentor, enterprise, requirement)  # 10%
    geo_time_score = score_geo_time(mentor, enterprise)  # 10%
    
    # 3. 加权计算
    final_score = (
        0.30 * industry_score +
        0.20 * skill_score +
        0.20 * trl_irl_score +
        0.10 * scale_score +
        0.10 * service_score +
        0.10 * geo_time_score
    )
    
    return {
        'final_score': final_score,
        'match_type': match_type,
        'breakdown': {
            'industry': industry_score,
            'skill': skill_score,
            'trl_irl': trl_irl_score,
            'scale': scale_score,
            'service': service_score,
            'geo_time': geo_time_score
        }
    }
```

#### 场景一：导师入驻与信息录入

**用户故事：** 作为导师，我希望快速录入我的专业信息，让平台了解我的能力，以便匹配到合适的企业。

```

    ▼            ▼               ▼               ▼              ▼
```

| 证据类型 | 录入方式 | 验证机制 | 对应TRL区间 |

## 八、总结

1. **15个交叉点类型**：实现更精准的匹配和定价
2. **六维度匹配模型**：前三维度权重占比70%

1. **创新的15类型分类**：取代传统的四等级分类，更精准

3. **可解释的匹配结果**：每个匹配都有明确的理由
4. **完整的产品路线图**：从MVP到规模化的清晰路径
