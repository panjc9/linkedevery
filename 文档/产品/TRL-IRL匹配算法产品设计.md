## 一、产品概述

### 1.1 产品定位

> **LinkedEvery 导师-企业智能匹配系统**是一个基于 TRL/IRL 方法论的导师-企业精准匹配平台，通过"诊断→处方→配药"三步法，为企业找到 TRL 领先其 IRL 1-2 级的最佳带教导师。

### 1.2 核心价值主张

| 维度 | 传统模式 | LinkedEvery |
|------|---------|------------|
| **匹配方式** | 靠关系、靠推荐、靠经验 | 基于 TRL/IRL 量化模型 |
| **匹配精度** | 模糊的"合不合适" | 精确的"差几级" |
| **定价逻辑** | 平台定价、一口价 | 公式定价、透明可解释 |
| **导师价值** | 一次性服务、无法沉淀 | 能力资产化、持续获益 |
| **企业体验** | 信息不对称、选择困难 | 匹配报告、可解释推荐 |

### 1.3 核心匹配公式

```
服务价格 = 基础价格 × 需求等级系数(L1-L4) × TRL稀缺性系数 × IRL差距系数

最佳带教距离 = 导师 TRL - 企业 IRL = 1~2 级
```

## 二、用户场景分析

### 2.1 用户角色定义

| 角色 | 典型用户 | 核心诉求 | 使用频率 |
|------|---------|---------|---------|
| **导师** | 高校教授、研究员、技术专家、产业顾问 | 展示能力、获取企业客户、持续获益 | 低频（首次录入+定期更新） |
| **企业** | 制造业企业、中小企业、数字化转型企业 | 找到合适的导师、解决技术难题 | 中频（有需求时使用） |
| **运营** | 平台运营人员、客服、BD | 管理导师/企业、审核匹配、处理异常 | 高频（日常运营） |

### 2.2 导师使用场景

#### 场景一：导师首次入驻

```
场景描述：
王教授是某高校智能制造方向的教授，有20年研究经验，
希望通过LinkedEvery平台对接制造业企业，将研究成果产业化。

用户旅程：
1. 收到邀请链接 / 看到平台宣传 → 访问平台
2. 使用手机号/微信注册账号
3. 进入"导师入驻"引导流程
4. 填写基本信息（姓名、院校、职称、研究方向）
5. 上传学术成果（论文、专利、项目经历）
   → 系统自动提取 TRL 等级（T1-T9）
6. 选择服务类型（研究型/技术型/工程型/服务型）
7. 设置服务偏好（行业偏好、地域偏好、时间偏好）
8. 确认并提交 → 等待审核
9. 审核通过 → 生成导师能力卡片（POS+POW区块）

用户目标：15分钟内完成入驻，展示自己的核心能力
痛点：
• 学术成果很多，不知道哪些对匹配有用
• 不清楚自己的 TRL 等级是多少
• 担心信息被泄露或滥用
```

#### 场景二：导师收到匹配推荐

```
场景描述：
王教授已入驻平台3天，收到一条匹配推荐通知。

用户旅程：
1. 收到微信/短信通知："有1家企业与您的能力高度匹配"
2. 点击通知 → 进入匹配详情页
3. 查看企业需求摘要（脱敏后的行业+需求等级+IRL等级）
4. 查看匹配报告：
   - 匹配度：95%
   - 匹配理由：企业IRL=3（试点），您的TRL=5（中试），差距+2，最佳带教距离
   - 预估收入：2.4倍基础价
5. 选择"感兴趣" / "暂不考虑"
6. 如感兴趣 → 双方进入对接流程

用户目标：快速判断是否值得投入时间
痛点：
• 企业需求描述不清，不知道具体要做什么
• 担心企业IRL太低，沟通成本高
• 不知道自己的服务该定什么价
```

#### 场景三：导师更新能力画像

```
场景描述：
王教授新发表了一篇顶级期刊论文，希望更新自己的能力画像。

用户旅程：
1. 登录平台 → 进入"我的能力画像"
2. 点击"更新学术成果"
3. 上传新论文 / 填写新项目经历
4. 系统自动重新评估 TRL 等级
5. 确认更新 → 版本归档（保留历史版本）

用户目标：让最新成果及时反映在匹配中
```

### 2.3 企业使用场景

#### 场景一：企业首次发布需求

```
场景描述：
某制造企业想引入AI质检，但不知道从哪开始，希望通过平台找到合适导师。

用户旅程：
1. 访问平台 → 注册企业账号
2. 进入"发布需求"引导流程
3. 第一步：描述需求（自然语言输入）
   → 输入："我们工厂想用AI做质检，但不知道怎么开始"
   → 系统NLP分析：A4 AI赋能 + L4研究型
4. 第二步：填写企业基本情况
   - 企业名称、行业、规模、年营收
   - 数字化现状（有无ERP/MES/CRM/IT团队）
5. 第三步：完成IRL评估问卷（15题，5分钟）
   → 系统自动评估：IRL=L2（原型阶段）
6. 第四步：确认需求画像
   → 系统生成需求卡片：
     • 技能方向：A4 AI赋能
     • 需求等级：L4 研究型
     • 企业IRL：L2（原型）
     • 推荐导师TRL：T4-T5
7. 提交需求 → 等待匹配结果

用户目标：5-10分钟完成需求发布，获得精准匹配结果
痛点：
• 不知道自己的需求属于什么等级
• 不清楚自己企业的数字化水平（IRL）
• 担心描述不清楚导致匹配不准
```

#### 场景二：企业查看匹配报告

```
场景描述：
企业发布需求后2小时内收到匹配报告。

用户旅程：
1. 收到通知："您的匹配报告已生成"
2. 点击进入匹配报告页
3. 查看报告内容：
   - 企业诊断：IRL=L2（原型阶段），数字化短板分析
   - 匹配结果：Top 3 导师推荐
   - 每个导师的：
     • 能力摘要（脱敏）
     • TRL等级
     • 匹配度评分
     • 匹配理由（可解释）
     • 预估服务价格
     • 服务形式和周期
4. 选择感兴趣的导师 → 发起对接请求
5. 等待导师确认 → 进入服务流程

用户目标：清楚了解为什么推荐这些导师，快速做出选择
```

#### 场景三：企业跟踪服务进展

```
场景描述：
企业已与导师对接，正在进行技术服务。

用户旅程：
1. 登录平台 → 进入"我的服务"
2. 查看当前服务状态：
   - 服务阶段：技术方案设计中
   - 已完成里程碑：需求确认 ✅、方案框架 ✅
   - 待完成里程碑：详细方案 ⏳、试点验证 ⬜
3. 查看导师提交的方案文档
4. 提交反馈和评价
5. 服务完成后 → 确认验收 → 释放付款

用户目标：实时了解服务进展，确保项目按计划推进
```

### 2.4 运营管理场景

#### 场景一：导师审核

```
场景描述：
运营人员需要审核新入驻的导师信息。

用户旅程：
1. 登录运营后台 → 待审核导师列表
2. 查看导师提交的信息：
   - 基本信息核实（院校、职称真实性）
   - 学术成果验证（论文查重、专利查询）
   - TRL等级复核（系统评估 vs 人工判断）
3. 操作：通过 / 驳回（附理由）/ 要求补充材料
4. 通过 → 导师能力卡片上线

运营目标：每天处理20+导师审核，确保信息真实可靠
```

#### 场景二：匹配质量监控

```
场景描述：
运营人员监控匹配质量，发现某类匹配效果不佳。

用户旅程：
1. 登录运营后台 → 匹配质量看板
2. 查看核心指标：
   - 匹配成功率（导师接受率 × 企业选择率）
   - 服务完成率
   - 双方满意度评分
   - 匹配度分布
3. 发现异常：L4研究型匹配成功率偏低（仅40%）
4. 分析原因：研究型需求的IRL评估偏差较大
5. 调整策略：优化IRL问卷中研究型相关的题目权重

运营目标：持续优化匹配质量，匹配成功率>70%
```

```
场景描述：
运营负责人查看平台整体运营数据。

2. 查看核心指标：
   - 导师数量：已入驻 156 人，本月新增 23 人
   - 企业数量：已注册 89 家，本月新增 12 家
   - 匹配次数：本月 45 次，成功率 72%
   - 交易金额：本月 ¥128,000
   - 需求等级分布：L1:15% L2:35% L3:30% L4:20%
   - TRL分布：T1-T3:25% T4-T6:45% T7-T9:30%

运营目标：掌握平台整体健康度，指导运营策略
```

```python
def auto_assess_trl(mentor_data):
    """
    根据导师填写的信息自动评估 TRL 等级
    返回：TRL等级(1-9) + 评估依据
    """
    scores = {
        'paper_score': 0,      # 论文贡献（T1-T3）
        'patent_score': 0,     # 专利贡献（T3-T5）
        'project_score': 0,    # 项目贡献（T4-T6）
        'industry_score': 0,   # 产业化贡献（T5-T9）
    }
    
    # 论文贡献评估
    paper_count = len(mentor_data.get('papers', []))
    if paper_count >= 20:
        scores['paper_score'] = 3
    elif paper_count >= 10:
        scores['paper_score'] = 2.5
    elif paper_count >= 5:
        scores['paper_score'] = 2
    elif paper_count >= 1:
        scores['paper_score'] = 1
    
    # 专利贡献评估
    patent_count = len(mentor_data.get('patents', []))
    invention_patents = sum(1 for p in mentor_data.get('patents', []) 
                          if p.get('type') == '发明专利')
    if invention_patents >= 3:
        scores['patent_score'] = 5
    elif invention_patents >= 1:
        scores['patent_score'] = 4
    elif patent_count >= 1:
        scores['patent_score'] = 3
    
    # 项目贡献评估
    projects = mentor_data.get('projects', [])
    national_projects = sum(1 for p in projects 
                          if '国家' in p.get('source', ''))
    provincial_projects = sum(1 for p in projects 
                            if '省' in p.get('source', ''))
    if national_projects >= 2:
        scores['project_score'] = 6
    elif national_projects >= 1:
        scores['project_score'] = 5
    elif provincial_projects >= 2:
        scores['project_score'] = 4.5
    elif provincial_projects >= 1:
        scores['project_score'] = 4
    
    # 产业化贡献评估
    industry_exp = mentor_data.get('industry_experience', [])
    if len(industry_exp) >= 3:
        scores['industry_score'] = 7
    elif len(industry_exp) >= 1:
        scores['industry_score'] = 5.5
    
    # 综合评估（取最高分，但考虑加权）
    final_trl = max(scores.values())
    
    # 特殊规则：如果产业化经验丰富，TRL不低于T5
    if len(industry_exp) >= 2 and final_trl < 5:
        final_trl = 5
    
    return {
        'trl': round(final_trl),
        'details': scores,
        'confidence': calculate_confidence(scores)
    }
```

```python
def analyze_enterprise_need(description):

    }

        'L4': ['不知道怎么开始', '不了解', '从0开始', '探索', '可行性', '调研', '评估'],
        'L3': ['技术搞不定', '需要技术', '开发', '攻关', '架构', '算法', '模型开发'],

        'L1': ['运维', '优化', '故障', '支持', '维护', '升级', '巡检'],
    }
    
    # 计算技能方向匹配度
    skill_scores = {}
    for skill, keywords in skill_keywords.items():
        score = sum(1 for kw in keywords if kw in description)
        skill_scores[skill] = score
    
    # 计算需求等级匹配度
    level_scores = {}
    for level, keywords in level_keywords.items():
        score = sum(1 for kw in keywords if kw in description)
        level_scores[level] = score
    
    # 取最高分
    best_skill = max(skill_scores, key=skill_scores.get)
    best_level = max(level_scores, key=level_scores.get)
    
    return {
        'skill_direction': best_skill,
        'need_level': best_level,
        'confidence': calculate_confidence(skill_scores, level_scores),
        'keywords': extract_keywords(description)
    }
```

#### 3.3.3 IRL 评估算法

```python
def assess_irl(questionnaire_answers):
    """
    IRL 评估算法
    输入：问卷答案
    输出：IRL等级(1-5) + 维度得分 + 短板分析
    """
    dimensions = {
        'it_infrastructure': 0,  # IT基础设施
        'data_foundation': 0,    # 数据基础
        'team_capability': 0,    # 团队能力
        'management_process': 0, # 管理流程
        'budget_resource': 0,    # 预算资源
    }
    
    # Q1: 信息化系统
    systems = questionnaire_answers.get('q1', [])
    system_count = len([s for s in systems if s != '无'])
    dimensions['it_infrastructure'] = min(system_count * 0.25, 1.0)
    
    # Q2: IT团队
    team_map = {'有（5人以上）': 1.0, '有（1-4人）': 0.7, '无（外包）': 0.4, '无': 0.1}
    dimensions['team_capability'] = team_map.get(questionnaire_answers.get('q2'), 0.1)
    
    # Q3: 数字化投入
    budget_map = {'<1%': 0.2, '1-3%': 0.5, '3-5%': 0.8, '>5%': 1.0}
    dimensions['budget_resource'] = budget_map.get(questionnaire_answers.get('q3'), 0.2)
    
    # Q4: 数据治理
    data_map = {'有，已体系化': 1.0, '正在建设': 0.7, '有但不完善': 0.5, '无': 0.1}
    dimensions['data_foundation'] = data_map.get(questionnaire_answers.get('q4'), 0.1)
    
    # Q5: AI应用
    ai_map = {'已落地（有实际应用）': 1.0, '试点中': 0.7, '计划中': 0.4, '无计划': 0.1}
    ai_score = ai_map.get(questionnaire_answers.get('q5'), 0.1)
    
    # Q6: 高层支持
    support_map = {'一把手工程': 1.0, '有分管领导': 0.8, '部门负责': 0.5, '无明确支持': 0.2}
    dimensions['management_process'] = support_map.get(questionnaire_answers.get('q6'), 0.2)
    
    # 综合评分
    avg_score = sum(dimensions.values()) / len(dimensions)
    
    # AI应用作为加分项
    final_score = avg_score * 0.8 + ai_score * 0.2
    
    # 映射到 IRL 等级
    if final_score >= 0.85:
        irl = 5
    elif final_score >= 0.70:
        irl = 4
    elif final_score >= 0.50:
        irl = 3
    elif final_score >= 0.30:
        irl = 2
    else:
        irl = 1
    
    # 识别短板
    weaknesses = []
    for dim, score in dimensions.items():
        if score < 0.4:
            weaknesses.append({
                'dimension': dim,
                'score': score,
                'suggestion': get_improvement_suggestion(dim, score)
            })
    
    return {
```

#### 3.4.2 匹配算法核心逻辑

```python
def match_mentors(enterprise_need, mentor_pool):
    """
    核心匹配算法
    输入：企业需求画像 + 导师池
    输出：Top N 匹配结果 + 匹配报告
    """
    results = []
    
    for mentor in mentor_pool:
        # Step 1: TRL/IRL 适配度评分
        trl_irl_score = calculate_trl_irl_fitness(
            mentor_trl=mentor['trl'],
            enterprise_irl=enterprise_need['irl'],
            need_level=enterprise_need['level']  # L1-L4
        )
        
        # Step 2: 技能匹配度评分
        skill_score = calculate_skill_fitness(
            mentor_skills=mentor['skills'],      # A1-A6
            required_skills=enterprise_need['skill_direction']
        )
        
        # Step 3: 行业匹配度评分
        industry_score = calculate_industry_fitness(
            mentor_industries=mentor['industries'],
            enterprise_industry=enterprise_need['industry']
        )
        
        # Step 4: 综合匹配度
        total_score = (
            trl_irl_score * 0.50 +    # TRL/IRL 权重50%
            skill_score * 0.30 +       # 技能匹配 权重30%
            industry_score * 0.20      # 行业匹配 权重20%
        )
        
        results.append({
            'mentor': mentor,
            'total_score': total_score,
            'trl_irl_score': trl_irl_score,
            'skill_score': skill_score,
            'industry_score': industry_score,
            'price_estimate': calculate_price(
                enterprise_need['level'],
                mentor['trl'],
                enterprise_need['irl']
            ),
            'match_reason': generate_match_reason(
                mentor, enterprise_need, trl_irl_score, skill_score, industry_score
            )
        })
    
    # 按匹配度排序，返回 Top N
    results.sort(key=lambda x: x['total_score'], reverse=True)
    return results[:3]


def calculate_trl_irl_fitness(mentor_trl, enterprise_irl, need_level):
    """
    TRL/IRL 适配度评分
    根据需求等级（L1-L4）采用不同的评分规则
    """
    gap = mentor_trl - enterprise_irl
    
    # 不同需求等级的最佳差距范围
    optimal_ranges = {
        'L4': (2, 4),   # 研究型：+2~+4 最佳
        'L3': (1, 3),   # 技术型：+1~+3 最佳
        'L2': (1, 2),   # 工程型：+1~+2 最佳
        'L1': (0, 2),   # 服务型：0~+2 最佳
    }
    
    optimal_min, optimal_max = optimal_ranges[need_level]
    
    if optimal_min <= gap <= optimal_max:
        return 100  # 最佳匹配
    elif gap == optimal_min - 1 or gap == optimal_max + 1:
        return 80   # 可用匹配
    elif gap == optimal_min - 2 or gap == optimal_max + 2:
        return 50   # 勉强可用
    else:
        return 20   # 不推荐


def calculate_price(need_level, mentor_trl, enterprise_irl):
    """
    定价计算
    """
    base_price = 1000  # 基础价格（元/小时）
    
    level_coefficients = {'L4': 3.0, 'L3': 2.0, 'L2': 1.2, 'L1': 1.0}
    
    # TRL稀缺性系数
    if mentor_trl <= 3:
        trl_coeff = 1.5
    elif mentor_trl <= 6:
        trl_coeff = 1.2
    else:
        trl_coeff = 1.0
    
    # IRL差距系数
    gap = mentor_trl - enterprise_irl
    if 1 <= gap <= 2:
        gap_coeff = 1.0
    elif gap == 0:
        gap_coeff = 0.8
    elif 3 <= gap <= 4:
        gap_coeff = 1.3
    else:
        gap_coeff = 0.5
    
    price = base_price * level_coefficients[need_level] * trl_coeff * gap_coeff
    
    return {
        'base_price': base_price,
        'level_coefficient': level_coefficients[need_level],
        'trl_coefficient': trl_coeff,
        'gap_coefficient': gap_coeff,
        'final_price': price,
        'price_multiplier': price / base_price
    }
```

### 4.3 操作便捷性设计

#### 4.3.1 快捷操作

```
快捷操作设计：

1. 一键匹配
   • 企业填写完需求后，一键触发匹配
   • 无需额外操作，2小时内收到报告

2. 快速反馈
   • 匹配报告中直接点击"感兴趣"/"暂不考虑"
   • 无需跳转，减少操作步骤

3. 智能推荐
   • 根据企业历史需求，推荐相似导师
   • 根据导师服务记录，推荐相似企业

4. 消息通知
   • 匹配结果通过微信/短信推送
   • 重要节点（审核通过、对接确认）实时通知

5. 批量操作
   • 运营端支持批量审核导师
   • 支持批量导出匹配报告
```

#### 4.3.2 响应式设计

```
响应式设计策略：

1. 移动端优先
   • 导师入驻：手机端完成80%操作
   • 企业需求：手机端完成100%操作
   • 匹配报告：手机端阅读+决策

2. PC端增强
   • 运营后台：PC端为主
   • 详细报告：PC端阅读体验更好
   • 数据分析：PC端图表展示

3. 关键页面适配
   • 导师入驻：手机端分步表单
   • IRL问卷：手机端单题展示
   • 匹配报告：手机端卡片式布局
```

### 7.2 定价策略

```
匹配服务费定价：

基础价格：¥5,000/次

最终价格 = 基础价格 × 需求等级系数 × TRL稀缺性系数 × IRL差距系数

示例：
• L4研究型 + 学术导师(T4) + 企业(IRL=1)
  = ¥5,000 × 3.0 × 1.5 × 1.0 = ¥22,500

• L3技术型 + 技术专家(T6) + 企业(IRL=3)
  = ¥5,000 × 2.0 × 1.2 × 1.0 = ¥12,000

• L2工程型 + 实施顾问(T7) + 企业(IRL=4)
  = ¥5,000 × 1.2 × 1.0 × 1.0 = ¥6,000

• L1服务型 + 运维工程师(T9) + 企业(IRL=5)
  = ¥5,000 × 1.0 × 1.0 × 0.8 = ¥4,000
```

### 8.1 核心风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|

| **导师信息造假** | 匹配失准 | 中 | 多维度验证，引入第三方数据源 |
| **企业IRL评估偏差** | 匹配不准 | 中 | 结合问卷+自动推断+人工校验 |
| **定价争议** | 交易纠纷 | 低 | 定价透明化，价格构成可解释 |
